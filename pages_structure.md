# Структура Страниц и Архитектура Интерфейсов Next.js 16

Этот документ описывает файловую структуру, организацию компонентов и архитектурные потоки данных для фронтенд-приложения игры «Бункер 2077». Все компоненты стилизованы в концепции **Tactical Post-Apocalyptic UI** с использованием Tailwind CSS v4 и Radix UI, исключая любые неоновые свечения и глитчи.

---

## 2. Потоки Управления и Состояния (UI State Management)

### Global State Context
1.  **SysAuthContext:** Распределяет сессии. Для гостевого входа генерируется уникальный UUID, который сохраняется в `localStorage` и соотносится с `Supabase Anonymous Auth`. Данные профилей игроков (например, позывной и список друзей) синхронизируются автоматически.
2.  **SysSoundContext:** Реализует глобальный Mute-переключатель. Воспроизводит аудиоэффекты (клики механического реле, аварийный басовый зуммер при падении таймера < 15с) через Web Audio API.
3.  **RealtimeLinkContext:** Постоянный websocket-канал Supabase Realtime Broadcast для прослушивания входящих сигналов связи (приглашений к игре) в фоне с возможностью их быстрого принятия или отклонения.

### Потоки переходов между интерфейсами
*   **Диспетчер -> Лобби:** При успешной авторизации или клике на «Войти как Гость» создается запись комнаты в Supabase, генерируется код доступа (формат `ROOM-XXXX`). Пользователь переходит на `/lobby/[roomId]`.
*   **Лобби -> Терминал Выживания:** Когда хост заканчивает комплектование команды (включая выбор психотипов ИИ-ботов) и все нажимают готовность, по сигналу Realtime Broadcast срабатывает роутер Next.js, переводя всех на `/game/[roomId]`.
*   **Терминал Выживания -> Финальный Лог:** После изгнания участников до лимита вместимости бункера (`N`), на серверной стороне или клиенте запускается финальная сборка данных выживших. Пользователи направляются на страницу `/epilogue/[roomId]`, где рендерится история выживания с помощью Pollinations LLM.

## 3. Детализация UI компонентов (Спецификация)

### Кнопка с блокировкой случайных нажатий (Hold-to-Confirm Button)
*   **Расположение:** Панель голосования, кнопка `«ИЗГНАТЬ В ПУСТОШЬ»`.
*   **Механика:** Требует удержания нажатой кнопки мыши или тач-экрана в течение 1500мс. Внутренний таймер измеряет время, а Framer Motion или CSS-анимация заполняет кнопку цветом `bg-rose-600` слева направо. При отпускании мыши до истечения 1500мс анимация сбрасывается к исходному состоянию `bg-rose-950/40`.
*   **A11y:** Дополнительно дублируется текстовое уведомление для программ чтения экрана: «Удерживайте Enter в течение 1.5 секунд для изгнания игрока».

### Компонент Бортового Журнала (Typewriter System)
*   **Механика:** Обрабатывает текст финальной истории ИИ посимвольно (через hook `useTypewriter` с интервалом `15ms`).
*   **Оптимизация:** Читатели могут пропустить эффект печати, нажав кнопку `[ SKIP LOG EFFECT ]`, которая моментально отменяет таймер и выводит весь сгенерированный текст.
*   **Звук:** Имитирует срабатывание физических реле старых ЭВМ. При выводе каждого символа воспроизводится короткий звуковой импульс (10мс).
