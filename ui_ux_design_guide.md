# Визуальный UI/UX Гайд: «Бункер 2077» — Tactical Post-Apocalyptic UI

Этот гайд описывает дизайн-систему, графический стиль, структуру экранов, сетку и анимации для веб-игры «Бункер 2077». Интерфейс разработан с акцентом на высокую разборчивость, строгую милитаристскую эргономику и отсутствие перегружающих визуальных эффектов (неонового свечения, глитчей и «киберпанк-салата»). Дизайн-система построена на готовых стандартных палитрах Tailwind CSS v4 и Radix UI, обеспечивая соответствие стандартам доступности Web Content Accessibility Guidelines (WCAG 2.1 AA).

---

## 1. Концепция и Визуальный Стиль

*   **Концепция:** Interface of Survivability (Интерфейс Живучести). Экраны стилизованы под терминал жизнеобеспечения автономного убежища и приборы тактического планирования.
*   **Эстетика:** Милитари-минимализм, жесткие конструктивные линии, матовые поверхности, плоские функциональные плашки, микро-границы и высокая информационная плотность. Никакого дешевого неонового ИИ-клише и размытых светящихся теней.
*   **Материалы и текстуры:**
    *   **Стекло бункера:** Полупрозрачные матовые плашки `zinc-900/85` с размытием `backdrop-blur-md` и тонкими рамками `border-zinc-800` для четкого разделения зон.
    *   **Разделители:** Тонкие линии `border-zinc-800/80` для структурирования информации без визуального шума.
    *   **Текстура экрана:** Легкая функциональная сетка (grid overlay) с едва заметной прозрачностью (opacity ~3%), имитирующая антибликовое покрытие промышленного монитора.

### Реализация контрастов и A11y (WCAG 2.1):
*   Основной текст (`zinc-100` на `zinc-950` или `zinc-900`) обеспечивает контрастность выше **10:1** (намного выше требуемого 4.5:1).
*   Интерактивные ярлыки (`emerald-400` или `sky-500` на `zinc-900`) протестированы на контрастность > **4.5:1**, гарантируя считываемость при тусклом освещении.
*   Все интерактивные элементы при фокусе клавиатуры подсвечиваются жесткой обводкой `ring-2 ring-sky-500 ring-offset-2 ring-offset-zinc-950`.

## 3. Типографика

*   **Интерфейсный шрифт (Шрифт по умолчанию):** `Inter` или `System-UI Sans-Serif`. Используется для всех кнопок, форм, навигации, карточек профиля и системных настроек. Обеспечивает эргономику чтения текстов на мобильных устройствах и десктопах.
*   **Терминальный шрифт (Шрифт данных):** `JetBrains Mono` или `Fira Code`. Используется для чата, реплик ИИ-ботов, цифр ресурсов (вода, еда, кислород), логов игры, таймера обратного отсчета и текста финального бортового журнала. Подразделяется на:
    *   *Заголовки терминала:* `font-mono tracking-wider uppercase font-bold text-zinc-100`
    *   *Значения счетчиков:* `font-mono tabular-nums text-emerald-500` (для соосности цифр)

### Экран 1. Главный экран / Авторизация (Tactical Grid Lobby)

Экран разделен на строго структурированные отсеки: левую панель входа и правую информационную панель статуса систем Бункера.

```
+--------------------------------------------------------------+
| [=] BUNKER CONTROL UNIT : v4.0.0            [COMM-LINK (5)] |
+--------------------------------------------------------------+
|                                                              |
|   +-- [ 1. БЫСТРЫЙ ДОСТУП ] --+   +-- [ 2. АНАЛИЗАТОР УГРОЗ ] -+
|   | Имя: [ Tactical_Surv_3 ]  |   | [ Pollinations.ai Art ]    |
|   |                           |   | Concept: Post-nuclear silo |
|   |  [ КНОПКА: ВОЙТИ В СЕТЬ ] |   | Threat Level: CRITICAL     |
|   +---------------------------+   +----------------------------+
|                                                              |
|   +-- [ 3. АВТОРИЗАЦИЯ ] -----+   +-- [ 4. МОНИТОРИНГ СЕТИ ] --+
|   | Почта: _________________  |   | Сервер: Node-DE-West       |
|   | Пароль: ________________  |   | Игроков онлайн: 4,029      |
|   |  [ КНОПКА: СИСТЕМНЫЙ ВХОД]|   | Пинг: 24 ms                |
|   +---------------------------+   +----------------------------+
+--------------------------------------------------------------+
```

#### Кнопка «Вход в 1 клик для Гостя»
*   **Логика:** Автогенерация строгих тактических позывных (например, `Sentry-01`, `Rescuer-X`, `Analyst-12`). При клике генерируется UUID, записывается в `localStorage` и через `Supabase Anonymous Auth` пользователь получает доступ в лобби без ввода паролей.
*   **Дизайн:** Поле ввода `bg-zinc-950 border border-zinc-800 text-zinc-100 focus:border-sky-500 focus:ring-1 focus:ring-sky-500 transition-all rounded-sm`.
*   Кнопка: `bg-zinc-800 border border-zinc-700 text-zinc-300 hover:bg-zinc-700 hover:text-zinc-100 active:bg-zinc-900 rounded-sm py-2 text-sm font-semibold transition-colors uppercase tracking-wider`.

#### Панель Связи (`Comm-Link Sidebar` / Друзья)
*   **Спецификация:** Выдвижная панель справа (`w-72`), перекрывающая второстепенный контент с жестким скольжением без инерции.
*   **Компоненты:**
    1.  **Форма связи:** Поле добавления друга по коду `ID: BK-XXXX`. Кнопка с иконкой `Plus` (без цветного кричащего неона).
    2.  **Список корреспондентов:**
        *   `emerald-500` индикатор — `CONNECTED` (Доступен вызов -> Кнопка `[ Вызвать ]`).
        *   `amber-500` индикатор — `IN GAME` (Показывает номер сессии).
        *   `zinc-600` индикатор — `DISCONNECTED`.
    3.  **Realtime-приглашения (Supabase Broadcast):** При отправке приглашения у друга на экране появляется системный баннер: `ALERT: INCOMING LINK REQUEST FROM BK-9182` с кнопками `ACCEPT` (Мягкий зеленый `emerald-600/90`) и `DECLINE` (Мягкий красный `rose-700/90`).

#### Выбор психотипа ИИ при добавлении бота
Для каждого ИИ-бота создается индивидуальная карточка с его позывным и типом мышления:
1.  **Панический (PanicBot):** Фокусируется на немедленном выживании, пишет короткими эмоциональными фразами, паникует из-за воды, подсвечивается рамкой `border-rose-800/40`.
2.  **Циничный (CynicalBot):** Оценивает полезность кандидатов исключительно по сухим показателям (здоровье, профессия), жестко критикует беженецев без навыков, подсвечивается рамкой `border-amber-800/40`.
3.  **Стратегический (StrategistBot):** Рассчитывает баланс ресурсов, ищет оптимальные связи между игроками (например, связку «хирург + больной»), подсвечивается рамкой `border-sky-800/40`.

#### Панель Ресурсов (Resource Gauges)
*   Индикаторы выполнены в виде простых горизонтальных шкал (`bg-zinc-800 rounded-sm h-3 overflow-hidden`).
*   Цвет заполнения жестко привязан к уровню опасности:
    *   От 50% до 100%: `bg-emerald-500`
    *   От 20% до 50%: `bg-amber-500`
    *   Ниже 20%: `bg-rose-600`
*   Никаких внешних свечений и градиентов — только сухие матовые полосы.

#### Сетка характеристик «Survivor Card»
*   7 ключевых характеристик персонажа представлены в виде вертикального списка с кнопками-переключателями.
*   **Framer Motion Card Flip / Reveal:**
    *   Рубашка карты: сплошная стальная плита `bg-zinc-900 border border-zinc-800`.
    *   При раскрытии карта разворачивается по оси Y с быстрым, четким затуханием (без медленного качания).
    *   Эффект перехода: `easeOut` длительностью `0.3s` для экономии времени игрока.

#### Быстрые тактические сообщения (`Quick Tactical Triggers`)
Вместо глитч-кнопок используются плоские чипы под текстовым полем ввода:
*   `[❓ CONTRADICT]` — отправка авто-тезиса о недостоверности слов цели.
*   `[⚠️ SECURE USEFULNESS]` — выделяет рамкой выбранную характеристику и публикует аргумент.
*   `[☠️ DESIGNATE RISK]` — маркирует игрока как угрозу и предлагает проголосовать против него.

#### Пульт Исключения (Elimination Panel)
*   Сетчатый список с кнопками выбора. Вокруг портрета выбранного игрока появляется тонкая красная рамка `border-rose-600`.
*   Кнопка действия **«ИЗГНАТЬ В ПУСТОШЬ»**:
    *   Требует удержания кнопки мыши в течение **1.5 секунд** для защиты от случайного клика.
    *   При удержании кнопка равномерно заполняется цветом `rose-600` слева направо. При отпускании мыши прогресс мгновенно обнуляется.

## 6. Сетка, Адаптивность и Стили компонентов

### Правила сетки (Responsive Rules)
*   **Desktop (>= 1280px):** 3-колонный интерфейс. Левая колонка — характеристики персонажа (`w-[28%]`), Центр — чат и логи (`w-[46%]`), Правая колонка — пульт голосования (`w-[26%]`).
*   **Tablet (768px - 1024px):** 2-колонный интерфейс. Чат дебатов занимает правую половину экрана. Карты выжившего и пульт голосования располагаются слева друг под другом.
*   **Mobile (< 768px):** 1-колонный вертикальный стек с переключением разделов через нижний фиксированный системный таб-бар: `[1: МИССИЯ]` | `[2: ЧАТ]` | `[3: ГОЛОСОВАНИЕ]`. Карты персонажа скроллятся горизонтально.

## 7. UX-Фичи, Интерактив и Доступность (A11y)

*   **Keyboard Navigation (WCAG 2.1):**
    *   Все интерактивные элементы доступны по клавише `Tab`.
    *   При фокусе элемент выделяется жесткой контрастной обводкой `sky-500` (для темных поверхностей) или `rose-500` (для опасных зон).
    *   Списки кандидатов можно перелистывать стрелками клавиатуры, а выбор подтверждать клавишами `Space` / `Enter`.
*   **Звуковое сопровождение (Mechanical Feedback):**
    *   Звуки отключаются глобально в шапке прибора переключателем `[ SOUND: ON/OFF ]`.
    *   Клики интерфейса: приглушенные механические щелчки переключателей.
    *   Аварии/Таймер: низкочастотный глухой зуммер (без пронзительного визга).
*   **Оптимизация Realtime и Состояния Загрузки:**
    *   Для аватаров и карт используются жесткие скелетоны `bg-zinc-850 animate-pulse` с прямоугольными гранями (минимальное скругление `rounded-sm`).
